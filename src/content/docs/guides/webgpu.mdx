# WebGPU Support

Ionyx provides first-class support for **WebGPU**, the next generation of graphics and compute for the web.

## Why WebGPU?

WebGPU offers significant performance improvements over WebGL, providing:

- Lower overhead and more direct access to GPU hardware.
- Better support for modern graphics features (Ray Tracing, Compute Shaders).
- Multithreading and predictable performance.

## Using WebGPU in Ionyx

WebGPU is enabled by default in all Ionyx templates. You can access it through the standard `navigator.gpu` API.

### Checking for Support

```javascript
async function initGPU() {
  if (!navigator.gpu) {
    console.error("WebGPU is not supported on this device/platform.");
    return;
  }

  const adapter = await navigator.gpu.requestAdapter();
  if (!adapter) {
    console.error("No appropriate GPU adapter found.");
    return;
  }

  const device = await adapter.requestDevice();
  // Start your GPU-accelerated magic!
}
```

## Template Integration

When you initialize a project using `ionyx init`, the provided templates (React, Vue, Svelte) include built-in WebGPU support checks and example boilerplate to get you started immediately.

## Performance Tips

- **Texture Management**: Use Ionyx's native file system APIs to load assets directly into GPU memory.
- **Compute Shaders**: Leverage the GPU for heavy computations that would normally block the UI thread.
- **Native Interop**: Coming soon - direct interop between Rust `wgpu` and the WebView's WebGPU context.
