# Deployment

Deploy your Ionyx applications to various platforms and distribution channels.

## Build Configuration

Before deploying, configure your build settings in `ionyx.config.toml`:

```toml
[build]
target = "production"
optimize = true
compress = true
strip = true
parallel = true

[app]
name = "My Application"
version = "1.0.0"
description = "A sample Ionyx application"
author = "Your Company"
website = "https://myapp.com"

[package]
identifier = "com.company.myapp"
category = "Productivity"
copyright = "© 2024 Your Company"
```

## Platform-Specific Deployment

### Windows

#### Building for Windows

```bash
# Build Windows executable
ionyx build --platform windows

# Build with installer
ionyx build --platform windows --installer

# Build with code signing
ionyx build --platform windows --sign --certificate-path ./cert.pfx --certificate-password "password"
```

#### Windows Installer Options

```toml
[build.windows]
installer_type = "msi"  # or "exe"
create_desktop_shortcut = true
create_start_menu_shortcut = true
run_after_install = false
allow_downgrades = false
```

#### Code Signing Setup

1. Obtain a code signing certificate from a trusted CA
2. Install the certificate on your build machine
3. Configure signing in your build process

```bash
# Test signing
ionyx build --platform windows --sign --test-sign

# Production signing
ionyx build --platform windows --sign --timestamp-server "http://timestamp.digicert.com"
```

#### Windows Store Deployment

```toml
[build.windows_store]
package_identity_name = "YourCompany.MyApp"
publisher = "CN=Your Company"
display_name = "My Application"
```

### macOS

#### Building for macOS

```bash
# Build macOS app bundle
ionyx build --platform macos

# Build with code signing
ionyx build --platform macos --sign --identity "Developer ID Application: Your Name (TEAM_ID)"

# Build for notarization
ionyx build --platform macos --notarize --apple-id "your@email.com" --apple-password "app-password"
```

#### macOS Configuration

```toml
[build.macos]
bundle_identifier = "com.company.myapp"
bundle_version = "1.0.0"
short_version_string = "1.0"
category = "public.app-category.productivity"
lsui_element = false
```

#### Notarization Process

1. Sign your application
2. Upload to Apple for notarization
3. Staple the notarization ticket

```bash
# Complete notarization workflow
ionyx build --platform macos --sign --notarize --staple
```

#### Mac App Store

```toml
[build.mac_app_store]
team_id = "YOUR_TEAM_ID"
apple_id = "your@email.com"
short_description = "A sample Ionyx application"
full_description = "A comprehensive sample application built with Ionyx framework"
support_url = "https://support.myapp.com"
privacy_url = "https://myapp.com/privacy"
```

### Linux

#### Building for Linux

```bash
# Build Linux executable
ionyx build --platform linux

# Build AppImage
ionyx build --platform linux --appimage

# Build Debian package
ionyx build --platform linux --deb

# Build RPM package
ionyx build --platform linux --rpm
```

#### Linux Package Configuration

```toml
[build.linux]
desktop_file = "./assets/myapp.desktop"
icon_file = "./assets/myapp.png"
categories = "Office;Productivity;"
mimetypes = "text/plain;application/json;"
```

#### AppImage Configuration

```toml
[build.linux.appimage]
desktop_integration = true
update_information = "gh-releases-zsync|your-org|your-repo|latest|AppImage-x86_64.AppImage.zsync"
```

## Distribution Channels

### Direct Distribution

#### Self-Hosting

```bash
# Create distribution package
ionyx package --format all --output ./dist

# Generate update manifests
ionyx generate-updates --output ./updates
```

#### Auto-Update Setup

```toml
[updates]
enabled = true
base_url = "https://updates.myapp.com"
check_interval = 86400  # 24 hours
auto_download = false
auto_install = false
```

### App Stores

#### Microsoft Store

1. Register as a Windows Developer
2. Create a Store account
3. Submit your application for certification

```toml
[build.microsoft_store]
store_id = "9NBLGGH4J3J8"
publisher_name = "Your Company"
privacy_policy = "https://myapp.com/privacy"
```

#### Mac App Store

1. Enroll in Apple Developer Program
2. Create App Store Connect account
3. Submit for review

```toml
[build.mac_app_store]
app_category = "public.app-category.productivity"
content_rating = "4+"
rating_reasons = "Infrequent/Mild Simulated Gambling"
```

#### Snap Store

```bash
# Build Snap package
ionyx build --platform linux --snap

# Upload to Snap Store
snapcraft upload --release stable myapp_1.0.0_amd64.snap
```

#### Flatpak

```bash
# Build Flatpak
ionyx build --platform linux --flatpak

# Upload to Flathub
flatpak-builder --repo=repo --force-clean build-dir com.company.myapp.json
```

## Continuous Integration

### GitHub Actions

```yaml
# .github/workflows/build.yml
name: Build and Release

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    strategy:
      matrix:
        platform: [windows, macos, linux]
    
    runs-on: ${{ matrix.platform }}
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
    
    - name: Setup Rust
      uses: actions-rs/toolchain@v1
      with:
        toolchain: stable
    
    - name: Install Ionyx
      run: npm install -g ionyx
    
    - name: Build Application
      run: ionyx build --platform ${{ matrix.platform }} --production
    
    - name: Upload Artifacts
      uses: actions/upload-artifact@v3
      with:
        name: ${{ matrix.platform }}-artifact
        path: dist/
```

### GitLab CI

```yaml
# .gitlab-ci.yml
stages:
  - build
  - deploy

build:
  stage: build
  script:
    - npm install -g ionyx
    - ionyx build --platform all --production
  artifacts:
    paths:
      - dist/
    expire_in: 1 week

deploy:
  stage: deploy
  script:
    - ionyx deploy --platform all
  only:
    - tags
```

## Security Considerations

### Code Signing

Always sign your applications:

```bash
# Windows
ionyx build --platform windows --sign --certificate-path ./cert.pfx

# macOS
ionyx build --platform macos --sign --identity "Developer ID Application"

# Linux (GPG)
ionyx build --platform linux --sign --gpg-key-id "your-key-id"
```

### Security Scanning

```bash
# Dependency vulnerability scan
ionyx security-scan

# Static analysis
ionyx static-analyze

# Runtime security check
ionyx security-audit
```

### Sandboxing

```toml
[security]
sandbox = true
network_access = "limited"
file_access = "restricted"
shell_access = false
camera_access = false
microphone_access = false
```

### Metrics Collection

```javascript
import { analytics } from '@ionyx/runtime';

analytics.track('app_launched', {
  version: app.version,
  platform: platform.name,
  timestamp: Date.now()
});

analytics.track('feature_used', {
  feature: 'file_export',
  duration: performance.now()
});
```

## Troubleshooting

### Common Deployment Issues

**Code signing failures:**
- Verify certificate validity
- Check certificate chain
- Ensure correct certificate password

**Package installation failures:**
- Verify system requirements
- Check package dependencies
- Validate file permissions

**Update mechanism failures:**
- Check update server connectivity
- Verify update manifest format
- Test rollback procedures

### Debugging Builds

```bash
# Verbose build output
ionyx build --verbose

# Debug build
ionyx build --debug --platform windows

# Clean build
ionyx clean && ionyx build
```

### Testing Deployments

```bash
# Test installation
ionyx test-install ./dist/myapp.exe

# Test update mechanism
ionyx test-update ./dist/myapp.exe

# Test uninstallation
ionyx test-uninstall ./dist/myapp.exe
```

## Best Practices

1. **Version Management**: Use semantic versioning
2. **Release Notes**: Maintain detailed release notes
3. **Rollback Strategy**: Always have a rollback plan
4. **Testing**: Test on all target platforms
5. **Documentation**: Provide clear installation instructions
6. **Support**: Set up user support channels
7. **Monitoring**: Monitor application health and usage
8. **Security**: Regularly update dependencies and scan for vulnerabilities
